ssh-audit is a tool for SSH server and client configuration auditing. It inspects supported authentication methods, key exchange algorithms, host key types, encryption ciphers, and MAC algorithms, and flags anything that is considered weak or deprecated.

Upstream project: [jtesta/ssh-audit](https://github.com/jtesta/ssh-audit)

## Usage

The container's entrypoint is `python3 ./ssh-audit.py`. Pass target arguments directly after the image name.

### Audit a remote SSH server

```bash
docker run --rm ghcr.io/jauderho/ssh-audit:latest <hostname>
```

### Audit a server on a non-standard port

```bash
docker run --rm ghcr.io/jauderho/ssh-audit:latest <hostname> -p 2222
```

### Show help

```bash
docker run --rm ghcr.io/jauderho/ssh-audit:latest --help
```

## Exposed ports

The image declares `EXPOSE 2222`. This is informational — ssh-audit connects **outbound** to the target you specify. You do not need to publish port 2222 unless you are using ssh-audit's built-in server listening mode.
