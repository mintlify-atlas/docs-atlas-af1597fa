This category covers tools for auditing and validating the security posture of SSH servers, TLS/SSL configurations, Docker hosts, and source code. Each image is built on a minimal base, runs as a non-root user where possible, and is pinned to a digest for reproducible pulls.

## Common hardening properties

All images in this group share the following properties:

- **Digest-pinned base images** — Base images are referenced by `sha256` digest, preventing silent tag mutations.
- **Non-root execution** — Processes run as unprivileged users (`nobody`, `ssh-audit`, `testssl`) where the tool supports it.
- **Minimal dependencies** — Multi-stage builds (where applicable) and `apk del` / `apt autoremove` remove build-time packages from the final image.
- **No shell in Go images** — The trufflehog image is built from a scratch-compatible Alpine layer with only the compiled binary and `git`.

## Registries

All images are available from two registries:

| Registry | Pull command prefix |
|---|---|
| GitHub Container Registry (recommended) | `ghcr.io/jauderho/<image>:latest` |
| Docker Hub | `jauderho/<image>:latest` |

<Note>
GitHub Container Registry is the primary build target and has higher pull rate limits than Docker Hub.
</Note>
