[OpenTofu](https://github.com/opentofu/opentofu) is the open-source, community-driven fork of Terraform, maintained by the [Linux Foundation](https://www.linuxfoundation.org/). It was created after HashiCorp relicensed Terraform from MPL-2.0 to the Business Source License (BSL). OpenTofu retains full compatibility with existing Terraform HCL configurations and state files while committing to an open-governance model.

This image is built from the OpenTofu source with CGO enabled (required for some providers) against a hardened Alpine runtime base. The binary is named `tofu`.

## Usage

The image sets `tofu` as its `ENTRYPOINT`. The workflow mirrors Terraform exactly: `init` → `plan` → `apply`.

### Basic workflow

```bash
docker run --rm \
  -v $(pwd):/workspace \
  -w /workspace \
  ghcr.io/jauderho/opentofu:latest \
  init
```

```bash
docker run --rm \
  -v $(pwd):/workspace \
  -w /workspace \
  ghcr.io/jauderho/opentofu:latest \
  plan
```

```bash
docker run --rm \
  -v $(pwd):/workspace \
  -w /workspace \
  ghcr.io/jauderho/opentofu:latest \
  apply -auto-approve
```

### Full sequence with a saved plan

```bash
# Init
docker run --rm -v $(pwd):/workspace -w /workspace \
  ghcr.io/jauderho/opentofu:latest init

# Plan and save
docker run --rm -v $(pwd):/workspace -w /workspace \
  ghcr.io/jauderho/opentofu:latest plan -out=tfplan

# Apply the saved plan
docker run --rm -v $(pwd):/workspace -w /workspace \
  ghcr.io/jauderho/opentofu:latest apply tfplan
```

### Check the version

```bash
docker run --rm ghcr.io/jauderho/opentofu:latest version
```

## Differences from the Terraform image

| Property | Terraform | OpenTofu |
|---|---|---|
| Binary name | `terraform` | `tofu` |
| Upstream source | `github.com/hashicorp/terraform` | `github.com/opentofu/opentofu` |
| License | Business Source License | Mozilla Public License 2.0 |
| Governance | HashiCorp / IBM | Linux Foundation (open) |
| CGO | Disabled | Enabled (musl, static) |
| Build flags | `-ldflags="-s -w"` | `-ldflags="-s -w -X .../version.dev=no"` |

<Note>
  OpenTofu state files and HCL configurations are compatible with Terraform. You can switch between the two images without modifying your `.tf` files or state.
</Note>
