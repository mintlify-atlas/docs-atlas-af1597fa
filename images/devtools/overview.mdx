This section covers hardened Docker images for infrastructure-as-code and automation tools. Each image is built from source using a multi-stage build, pinned to a digest-verified base image (`ghcr.io/jauderho/alpine`), and published to both GitHub Container Registry and Docker Hub.

## Common traits

All images in this group share the following hardening properties:

- **Multi-stage builds** — build toolchain is discarded; only the final binary is copied into the runtime image.
- **Pinned digests** — every `FROM` instruction references a `sha256` digest, eliminating tag-mutability risks.
- **Minimal base** — runtime images use `ghcr.io/jauderho/alpine`, which is kept up to date with `apk upgrade -a` at build time.
- **Dual registry** — images are pushed to both `ghcr.io/jauderho/<tool>` and `docker.io/jauderho/<tool>`.
- **Multi-platform** — images are built for `linux/amd64`, `linux/arm64`, and `linux/s390x`.
- **SBOM + provenance** — each build attaches a Software Bill of Materials and build provenance attestation.
