  The upstream `nginx-quic` Mercurial branch is no longer actively maintained. QUIC and HTTP/3 support has been merged into the mainline nginx release. Use the standard [nginx (hardened)](/images/web/nginx) image instead — it already includes `--with-http_v3_module` and full HTTP/3 support.
</Warning>

This image was the original vehicle for experimenting with QUIC in nginx before the feature was stabilized in mainline. It is preserved here for reference and legacy compatibility.

## What is QUIC/HTTP3?

QUIC is a transport protocol built on UDP that underpins HTTP/3. Unlike TCP-based HTTP/1.1 and HTTP/2, QUIC provides:

- **Connection multiplexing without head-of-line blocking** — each stream is independent; a lost packet stalls only that stream, not the entire connection.
- **Zero-RTT connection resumption** — returning clients can send data immediately without waiting for a TLS handshake.
- **Built-in encryption** — QUIC always uses TLS 1.3; unencrypted QUIC does not exist.
- **Connection migration** — connections survive IP address changes (useful for mobile clients switching between Wi-Fi and cellular).

<Note>
  QUIC runs over **UDP**, not TCP. Firewalls and load balancers that block UDP traffic on port 443 will silently fall back to HTTP/2 for clients that advertise HTTP/3 support via the `Alt-Svc` response header. Ensure UDP 443 is open end-to-end.
</Note>

## Run

Because QUIC uses UDP, you must publish port 443 for both TCP and UDP:

```bash
docker run -d \
  --name nginx-quic \
  -p 80:80 \
  -p 443:443/tcp \
  -p 443:443/udp \
  -v /path/to/nginx.conf:/etc/nginx/nginx.conf:ro \
  -v /path/to/certs:/etc/nginx/certs:ro \
  ghcr.io/jauderho/nginx-quic:latest
```

## Exposed ports

| Port | Protocol | Purpose |
|---|---|---|
| 80 | TCP | HTTP |
| 443 | TCP | HTTPS / HTTP/2 |
| 443 | **UDP** | QUIC / HTTP/3 |

## How it is built

The image uses a two-stage multi-architecture build:

**Stage 1 — BoringSSL + nginx-quic:**

```dockerfile
FROM ghcr.io/jauderho/golang:1.26.1-alpine3.23 AS build

# Build BoringSSL
RUN git clone --depth 1 https://boringssl.googlesource.com/boringssl && \
    mkdir boringssl/build && \
    cd boringssl/build && cmake .. && make

# Clone and build nginx-quic
RUN hg clone https://hg.nginx.org/nginx-quic && \
    cd nginx-quic && hg update quic && \
    auto/configure \
      --with-http_v3_module \
      --with-http_ssl_module \
      --with-http_v2_module \
      --with-cc-opt='-I/src/boringssl/include ...' \
      --with-ld-opt='-L/src/boringssl/build/ssl -L/src/boringssl/build/crypto ...' && \
    make && strip objs/nginx && make install
```

**Stage 2 — minimal Alpine runtime:**

```dockerfile
FROM ghcr.io/jauderho/alpine:3.23.3

COPY --from=build /src/nginx-quic/objs/nginx /usr/sbin
COPY --from=build /src/files/nginx.conf /etc/nginx/nginx.conf

EXPOSE 80 443
ENTRYPOINT ["nginx", "-g", "daemon off;"]
```

## Enabling HTTP/3 in nginx.conf

A minimal configuration to serve HTTP/3 alongside HTTP/2:

```nginx
server {
    # HTTP/2 over TCP
    listen 443 ssl;
    listen [::]:443 ssl;

    # HTTP/3 over QUIC/UDP
    listen 443 quic reuseport;
    listen [::]:443 quic reuseport;

    ssl_certificate     /etc/nginx/certs/fullchain.pem;
    ssl_certificate_key /etc/nginx/certs/privkey.pem;
    ssl_protocols       TLSv1.3;

    # Advertise HTTP/3 to clients
    add_header Alt-Svc 'h3=":443"; ma=86400';

    location / {
        root  /usr/share/nginx/html;
        index index.html;
    }
}
```

<Tip>
  The `Alt-Svc` response header tells supporting browsers that HTTP/3 is available on port 443. On the next visit, the browser will attempt QUIC first and fall back to TCP if UDP is blocked.
</Tip>
