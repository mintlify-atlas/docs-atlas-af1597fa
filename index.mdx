## Get started in seconds

<Steps>
  <Step title="Choose a registry">
    Images are available on GitHub Container Registry (recommended) and Docker Hub.

    ```bash
    # GitHub Container Registry (recommended — higher rate limits)
    docker pull ghcr.io/jauderho/nginx:latest

    # Docker Hub
    docker pull jauderho/nginx:latest
    ```
  </Step>
  <Step title="Run your image">
    Every image follows standard Docker conventions. Most accept the same flags as the upstream tool.

    ```bash
    docker run --rm ghcr.io/jauderho/trufflehog:latest \
      github --repo https://github.com/example/repo
    ```
  </Step>
  <Step title="Pin to a version">
    Use version tags or digest references for reproducible deployments.

    ```bash
    # Version tag
    docker pull ghcr.io/jauderho/terraform:v1.12.1

    # Digest reference (most reproducible)
    docker pull ghcr.io/jauderho/terraform@sha256:<digest>
    ```
  </Step>
</Steps>

## Why these images?

<CardGroup cols={2}>
  <Card title="Security hardened" icon="shield-check">
    Compiled with `-fstack-protector-strong`, `-D_FORTIFY_SOURCE=3`, full RELRO. Non-root execution by default.
  </Card>
  <Card title="Automatically updated" icon="rotate">
    Upstream repos are monitored. New images build within 1 hour of an upstream release tag.
  </Card>
  <Card title="Daily CVE scanning" icon="bug">
    Every image is scanned daily by Trivy and Anchore. Build status is public.
  </Card>
  <Card title="Signed & SBOM included" icon="file-certificate">
    Images are signed with cosign. SBOMs are attached to every build for full provenance.
  </Card>
  <Card title="Minimal footprint" icon="minimize">
    Multi-stage builds leave build tools behind. Only the binary and runtime deps ship.
  </Card>
  <Card title="Multi-arch" icon="microchip">
    Most images support linux/amd64, linux/arm64, linux/ppc64le, and linux/arm/v7.
  </Card>
</CardGroup>
