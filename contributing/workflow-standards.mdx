---
title: "GitHub Actions Workflow Standards"
description: "Requirements for CI/CD workflows that build and publish images"
---

Every image in this repository is built and published through a dedicated GitHub Actions workflow. All workflows must conform to the standards described on this page. The canonical reference is `.github/workflows/adguardhome.yml`.

---

## File Location and Naming

Create the workflow at:

```
.github/workflows/<program>.yml
```

The workflow `name:` field must match the program directory name exactly — this value is used in the image tag via `${{ github.workflow }}`.

---

## Full Workflow Structure

The complete `adguardhome.yml` workflow is shown below. Use this as the starting point for any new workflow:

```yaml
name: adguardhome

on:
  push:
    branches: 
      - main
    paths:
      - adguardhome/*
      - .github/workflows/adguardhome.yml
  workflow_dispatch:
  schedule:
    - cron: "0 8 * * 1"

env:
  BUILD_VERSION: "v0.108.0-b.71"
  DOCKER_CLI_EXPERIMENTAL: enabled
  REPOSITORY: ${{ github.actor }}/${{ github.workflow }} 

permissions: read-all

jobs:
  deploy:
    runs-on: ubuntu-24.04
    permissions:
      packages: write

    steps:
      - name: Harden Runner
        uses: step-security/harden-runner@fe104658747b27e96e4f7e80cd0a94068e53901d # v2.16.1
        with:
          egress-policy: audit
          disable-telemetry: false
          allowed-endpoints: 
            95s5acprodeus1file6.blob.core.windows.net:443
            api.github.com:443
            artifactcache.actions.githubusercontent.com:443
            auth.docker.io:443
            codeload.github.com:443
            dl-cdn.alpinelinux.org:443
            docker-images-prod.6aa30f8b08e16409b46e0173d6de2f56.r2.cloudflarestorage.com:443
            ghcr.io:443
            github.com:443
            pkg-containers.githubusercontent.com:443
            production.cloudflare.docker.com:443
            proxy.golang.org:443
            registry-1.docker.io:443
            sum.golang.org:443
          
      - name: Source checkout
        uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd # v6.0.2
 
      - name: Setup QEMU
        id: qemu
        uses: docker/setup-qemu-action@ce360397dd3f832beb865e1373c09c0e9f86d70a # v4.0.0
      
      - name: Setup Buildx
        id: buildx
        uses: docker/setup-buildx-action@4d04d5d9486b7bd6fa91e7baf45bbb4f8b9deedd # v4.0.0

      - name: Set Docker metadata
        id: docker_meta
        uses: docker/metadata-action@030e881283bb7a6894de51c315a6bfe6a94e05cf # v6.0.0
        with:
          images: ${{ env.REPOSITORY }}
          labels: |
            org.opencontainers.image.version=${{ env.BUILD_VERSION }}
            org.opencontainers.image.revision=${{ github.sha }}
            org.opencontainers.image.title=${{ env.REPOSITORY }}

      - name: GitHub login
        if: ${{ github.event_name != 'pull_request' }}
        uses: docker/login-action@4907a6ddec9925e35a0a9e82d7399ccc52663121 # v4.1.0
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: DockerHub login
        if: ${{ github.event_name != 'pull_request' }}
        uses: docker/login-action@4907a6ddec9925e35a0a9e82d7399ccc52663121 # v4.1.0
        with:
          username: ${{ secrets.DOCKERHUB_USERNAME }}
          password: ${{ secrets.DOCKERHUB_PASSWORD }}      

      - name: Build and push
        uses: docker/build-push-action@d08e5c354a6adb9ed34480a06d141179aa583294 # v7.0.0
        with:
          push: ${{ github.event_name != 'pull_request' }}
          context: ${{ github.workflow }}
          platforms: linux/amd64,linux/arm64,linux/ppc64le,linux/arm/v7
          build-args: |
            BUILD_VERSION
          sbom: true
          provenance: true
          cache-from: type=gha, scope=${{ github.workflow }}
          cache-to: type=gha, scope=${{ github.workflow }}
          labels: ${{ steps.docker_meta.outputs.labels }}
          tags: |
            docker.io/${{ env.REPOSITORY }}:${{ env.BUILD_VERSION }}
            docker.io/${{ env.REPOSITORY }}:latest
            ghcr.io/${{ env.REPOSITORY }}:${{ env.BUILD_VERSION }}
            ghcr.io/${{ env.REPOSITORY }}:latest
```

## Requirements Breakdown

### Triggers

All three trigger types are required:

<Steps>
  <Step title="push (path-filtered)">
    The `push` trigger must be filtered to only run when files under the program's directory or its own workflow file change:

    ```yaml
    on:
      push:
        branches: 
          - main
        paths:
          - adguardhome/*
          - .github/workflows/adguardhome.yml
    ```

    Replace `adguardhome` with your program name in both path entries.
  </Step>

  <Step title="workflow_dispatch">
    Allows the workflow to be triggered manually from the GitHub Actions UI. No additional configuration needed:

    ```yaml
      workflow_dispatch:
    ```
  </Step>

  <Step title="Weekly schedule">
    Runs automatically on a weekly schedule to pick up base image updates and upstream changes even when no code is pushed. The `adguardhome` example runs every Monday at 08:00 UTC:

    ```yaml
      schedule:
        - cron: "0 8 * * 1"
    ```
  </Step>
</Steps>

### Security Requirements

<Warning>
  Both of these security requirements are mandatory. Pull requests that omit either will not be accepted.
</Warning>

#### Harden Runner

`step-security/harden-runner` must be the **first step** in every job. It monitors and restricts outbound network traffic during the build:

```yaml
- name: Harden Runner
  uses: step-security/harden-runner@fe104658747b27e96e4f7e80cd0a94068e53901d # v2.16.1
  with:
    egress-policy: audit
    disable-telemetry: false
    allowed-endpoints: 
      api.github.com:443
      ghcr.io:443
      github.com:443
      registry-1.docker.io:443
      # ... add any additional endpoints your build requires
```

Add every external host your build contacts to `allowed-endpoints`. For Go builds, include `proxy.golang.org:443` and `sum.golang.org:443`.

#### Pin all actions to full commit hashes

Every `uses:` reference must be pinned to a full 40-character commit SHA. Include a comment with the human-readable version tag for maintainability:

```yaml
# Correct
uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd # v6.0.2

# Wrong — tag references are mutable
uses: actions/checkout@v4
```

This applies to every action used in the workflow, including `docker/*` actions and `step-security/harden-runner`.

### Environment Variables

Hardcode `BUILD_VERSION` in the top-level `env` section. This is the version of the upstream tool being built:

```yaml
env:
  BUILD_VERSION: "v0.108.0-b.71"
  DOCKER_CLI_EXPERIMENTAL: enabled
  REPOSITORY: ${{ github.actor }}/${{ github.workflow }}
```

`BUILD_VERSION` is passed to the Docker build via `build-args` and used to version the image tags. Update it when you want to build a new upstream release.

---

### Registry Publishing

The workflow must push to **both** registries. Login steps are guarded by `if: ${{ github.event_name != 'pull_request' }}` to avoid pushing from PRs:

```yaml
- name: GitHub login
  if: ${{ github.event_name != 'pull_request' }}
  uses: docker/login-action@4907a6ddec9925e35a0a9e82d7399ccc52663121 # v4.1.0
  with:
    registry: ghcr.io
    username: ${{ github.actor }}
    password: ${{ secrets.GITHUB_TOKEN }}

- name: DockerHub login
  if: ${{ github.event_name != 'pull_request' }}
  uses: docker/login-action@4907a6ddec9925e35a0a9e82d7399ccc52663121 # v4.1.0
  with:
    username: ${{ secrets.DOCKERHUB_USERNAME }}
    password: ${{ secrets.DOCKERHUB_PASSWORD }}
```

Both versioned and `latest` tags are pushed to each registry:

```yaml
tags: |
  docker.io/${{ env.REPOSITORY }}:${{ env.BUILD_VERSION }}
  docker.io/${{ env.REPOSITORY }}:latest
  ghcr.io/${{ env.REPOSITORY }}:${{ env.BUILD_VERSION }}
  ghcr.io/${{ env.REPOSITORY }}:latest
```

### Multi-Platform Builds

Builds must target all four supported platforms:

```yaml
platforms: linux/amd64,linux/arm64,linux/ppc64le,linux/arm/v7
```

QEMU and Buildx setup steps are required to support cross-platform builds:

```yaml
- name: Setup QEMU
  id: qemu
  uses: docker/setup-qemu-action@ce360397dd3f832beb865e1373c09c0e9f86d70a # v4.0.0

- name: Setup Buildx
  id: buildx
  uses: docker/setup-buildx-action@4d04d5d9486b7bd6fa91e7baf45bbb4f8b9deedd # v4.0.0
```

<Note>
  If a tool's upstream does not support a particular architecture, that platform may be omitted. The `adguardhome.yml` workflow shows commented-out alternatives as reference for common reduced platform sets.
</Note>

### SBOM and Provenance

Software Bill of Materials (SBOM) and provenance attestations must be enabled on the build-and-push step:

```yaml
- name: Build and push
  uses: docker/build-push-action@d08e5c354a6adb9ed34480a06d141179aa583294 # v7.0.0
  with:
    sbom: true
    provenance: true
    # ...
```

These are generated automatically by `docker/build-push-action` and attached to the image manifest.
